# get started

## prerequisites

- **Ruby + Rake** (the command entry point; already present if you work with Jekyll/CollectionBuilder)
- **Node.js 18+** (runs the embedding pipeline; install the LTS release from <https://nodejs.org/>)

Dependencies are installed with npm (bundled with Node) or pnpm — `rake setup` finds whichever is available.

## quickstart

1. Review `config.yml` in the repository root and pick a model and preprocessing profile (the demo defaults, `clip` + `binary`, are recommended when users will upload photos of physical marks to search a collection of high-contrast scans).

2. Install JavaScript dependencies:

```sh
rake setup
```

3. Build the search data (the first run downloads model weights, which are cached for later runs):

```sh
rake build_embeddings
```

4. Serve the demo site and open it:

```sh
rake serve
```

Then open: <http://127.0.0.1:4173/>

Any static server works, e.g. `python3 -m http.server 4173 --directory docs`.

## configuration

`config.yml` controls the whole pipeline:

| setting | values | notes |
|---|---|---|
| `model` | `clip` (demo default), `dinov2`, `mobilenet` | which embedding model to use; see README for trade-offs |
| `preprocessing` | `binary` (demo default), `lineart`, `standard` | image normalization applied identically at build time and to user uploads |
| `objects_dir` | path | collection images |
| `metadata_csv` | path | metadata spreadsheet |
| `filename_field` | CSV column name | joins metadata rows to image files |
| `output_dir` | path | where artifacts are written (the page expects `data/` next to itself) |
| `top_k` | integer | how many results the page shows |

The build exports the resolved configuration to `docs/data/manifest.json`, and the search page configures itself from that manifest — after changing `config.yml`, just re-run `rake build_embeddings`. No page edits are needed.

## generated artifacts

After `rake build_embeddings`, files are written to `docs/data/`:

- `manifest.json` — build configuration export: model, preprocessing profile, quantization parameters, ordered filename list, score calibration
- `embeddings.bin` — collection vectors: `count × dim` int8 values (L2-normalized, scale 127), row-major; row *i* belongs to `manifest.filenames[i]`
- `index.json` — trimmed metadata records for the result cards (empty fields omitted)
- `build-info.json` — counts, skipped images, unmatched metadata rows
- `preprocess.log` — plain-text build summary

## smoke test checklist

1. Run the build:

```sh
rake setup
rake build_embeddings
```

Expected summary for the demo collection:

```text
Summary: images_found=196, embeddings_built=195, skipped_images=1
```

The one skip is `10058.png` ("missing metadata row for filename") — currently expected.

2. Check `build-info.json`: `counts.embeddings_built` is 195 and `skipped` lists only `10058.png`.

3. Launch the site (`rake serve`) and open <http://127.0.0.1:4173/>. The intro card should name the model from your `config.yml` and its approximate download size. Click **Start image search** and wait for the status line to report **Ready**.

4. **Parity canary**: upload an image that is *in* the collection (e.g. `docs/objects/10060.png`). It must come back as the #1 result with a match score near 100%. This single check verifies that build-time and in-browser embeddings agree; if the top result is not the uploaded image, the pipeline and page are out of sync.

5. Upload a non-collection image and confirm ranked results render with similarity badges and metadata.

## evaluating models on your collection

The eval harness measures how reliably each model retrieves a collection image from a perturbed copy of itself (simulated crop, rotation, and photo degradation):

```sh
cd scripts
node eval_retrieval.mjs                       # full matrix: 3 models × 2 profiles
node eval_retrieval.mjs --models dinov2,clip --profiles lineart --sample 40
```

Use it to pick the best `config.yml` settings for your own material. Results for the demo collection are in the README.

## troubleshooting

- **`rake setup` says Node is missing or too old** — install the LTS release from <https://nodejs.org/> (18 or newer), reopen your terminal, re-run.
- **Slow first `rake build_embeddings`** — the model weights (~25–90 MB depending on model) are downloaded once and cached; later runs are fast.
- **Page says "collection was built with … but this page expects …"** — the data in `docs/data/` was generated by an older/different registry than the page code. Re-run `rake build_embeddings`.
- **Page status stuck on "Downloading model"** — the browser fetches model weights from the Hugging Face CDN on first visit (cached afterwards); check the network connection and the browser console.
- **Images skipped during build** — see `docs/data/preprocess.log` for per-file reasons (usually a missing metadata row or an unreadable file).
